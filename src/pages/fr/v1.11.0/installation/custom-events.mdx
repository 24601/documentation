Les événements personnalisés sont excellents pour ajouter plus d'intelligence en enregistrant des événements spécifiques au domaine avec les relectures de session. OpenReplay utilise 2 types d'événements: fonctionnels (`event`) et techniques (`issue`). Tous les événements sont indexés pour une recherche facile et synchronisés avec les enregistrements de session.

<Aside type="tip">
Nous avons également un tutoriel détaillé sur la façon d'envoyer des événements personnalisés, vous pouvez le consulter [ici](/tutorials/custom-events).
</Aside>

## Événements fonctionnels

Les événements fonctionnels sont indexés et facilitent la recherche des enregistrements de session. S'ils ont été reçus avec succès par OpenReplay, ils seront disponibles en tant que filtres dans la barre de recherche omnisearch.

![Événement fonctionnel](/static/custom-events-1.png#center)

### Avec NPM

Utilisez la méthode `event` pour signaler des événements fonctionnels tels que *commande terminée* ou *produit ajouté*. Il prend 2 paramètres: *nom* (chaîne) et *charge utile* (objet JS non récursif).


```js
tracker.event('product_added', 'shoes'); // after tracker.start()
```

## Événements techniques

Les événements techniques sont affichés dans la relecture de session sous l'onglet *Événements* dans DevTools et sous forme d'annotations sur la lecture. Ils sont également pris en compte dans les entonnoirs pour corréler les baisses de conversion avec les problèmes techniques.

![Événement technique](/static/custom-events-2.png#center)

### Avec NPM

`issue` est utilisé pour envoyer des événements techniques, tels que des erreurs, qui peuvent survenir dans votre pile ou dans d'autres systèmes en aval. `issue` prend 2 paramètres: *nom* (chaîne) et *charge utile* (objet JS non récursif).


```js
tracker.issue('payment_error', {code: 500, context: 42}); // after tracker.start()
```
