Il est assez simple d'obtenir le suiveur pour une application Vue.

## Ajouter le code de suivi

Bien que vous puissiez [obtenir une version copier-coller du code directement à partir de la plate-forme lorsque vous créez un nouveau projet](/using-or/snippet#getting-the-snippet), ce code est trop basique et rendra la personnalisation et l'ajout de plugins supplémentaires beaucoup plus difficiles.

Si vous construisez une SPA (Single Page Application), tout ce que vous avez à faire est d'ajouter le code suivant à votre script de configuration dans votre fichier principal.


```html
<script setup>
//... your code here
const tracker = new Tracker({
  projectKey: "<your project key>"
});

tracker.start();
//... more code here
</script>
```


<Aside type="tip">
**Note pour les utilisateurs hébergés en local**: si vous utilisez la version hébergée en local d'OpenReplay, vous devrez également spécifier la propriété de configuration `ingestPoint`.
Cette propriété spécifie le point d'ingestion des données du suiveur. Les utilisateurs du Cloud n'ont pas besoin de cela,
car par défaut, le suiveur saura où se trouve le point d'accès du SaaS, mais si vous l'hébergez en local, vous devrez le spécifier
(ce devrait être quelque chose comme `https://openreplay.mydomain.com/ingest`)
</Aside>

### Gérer le "projectKey" dans votre code

Notez que la `projectKey` que vous configurez est celle fournie par la plate-forme OpenReplay.
Cette valeur ne doit pas être stockée en dur dans votre code pour des raisons de sécurité, mais plutôt dans un fichier/système de configuration.

Une option serait d'ajouter une variable d'environnement dans un fichier `.env` à la racine de votre projet et de préfixer la variable avec `VITE_`, ce qui la rendra disponible dans votre code Vue.
Vous pouvez l'utiliser plus tard en utilisant `import.meta.env.<votre nom de variable>`.
Comme ceci:


```javascript
const tracker = new Tracker({
  projectKey: import.meta.env.VITE_VUE_APP_PROJECT_KEY,
});
 ```
