Si les informations par défaut suivies par notre suiveur lorsqu'une erreur JS se produit ne sont pas suffisantes, vous pouvez utiliser cette méthode.

La méthode `handleError` signale manuellement les exceptions capturées, les promesses rejetées ou les événements d'erreur.

Voir [rapport d'erreur](/installation/error-reporting#manuallyloggingexceptions) pour plus de détails.

## Signature

```js 
handleError: (e: Error | ErrorEvent | PromiseRejectionEvent, metadata?: Record<string, any>) => void
```
