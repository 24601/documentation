Этот метод можно использовать для сообщения о «техническом событии» («/installation/custom-events»), иначе известном как «issue».
Эти проблемы могут использоваться для сообщения дополнительной информации о проблеме, которую отслеживатель не может подобрать по умолчанию. Хорошим примером является попытка пользователя оплатить покупку на нашем электронном магазине и получение ошибки от шлюза платежей. Хотя отслеживатель мог бы записать пакет с ошибкой, он не был бы идентифицирован как проблема. Этот метод позволяет пометить это поведение как проблему для дальнейшего фильтрования сессий.

<Aside type="tip">
У нас также есть подробное руководство по отслеживанию пользовательских событий, вы можете проверить его [здесь] (/tutorials/custom-events).
</Aside>

## Подпись

```js
issue(key: string, payload?: any): void
```
