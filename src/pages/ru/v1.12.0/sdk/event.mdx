Этот метод позволяет отчитываться о «функциональном событии». С помощью функциональных событий вы можете сигнализировать моменты в сеансе пользователя, которые имеют функциональное значение и которые по умолчанию наш трекер не может обнаружить и пометить самостоятельно.
Пример функционального события - добавление товара в корзину пользователем. Хотя наш трекер сможет записать это действие, он не сможет пометить его как соответствующее событие в таймлайне вашего сеанса.
Однако, благодаря этому методу вы можете сделать это вручную.

См. [пользовательские события] ( / installation / custom-events # functionalevents) для получения дополнительной информации.
