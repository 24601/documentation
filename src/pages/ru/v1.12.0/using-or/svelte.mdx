Установка трекера OpenReplay в проект на основе Svelte довольно проста.

Ключевой момент для работы будет использование контекста, который, хотя и похож на то, как мы делаем это с Next.js, но реализация будет отличаться.

## Пример репозитория

Если вы хотите просмотреть полный код для этого проекта, проверьте [репозиторий, который мы будем использовать в этом руководстве] (https://github.com/deleteman/openreplay-svelte-example).

Это пример приложения SvelteKit, которое вы можете клонировать и протестировать самостоятельно, следуя инструкциям в файле Readme.

После развертывания и настройки вы должны быть в состоянии захватить повторы сеансов, как показано на снимке ниже:

![Повтор сеанса Svelte] (images/svelte-session.png)

## Использование контекста для обмена данными

Это решение будет использовать контекст Svelte, который позволяет обмениваться данными между родительским компонентом и всеми его дочерними (похоже на контекст React).

В этом конкретном примере мы добавим трекер для раздела Администрирование нашего приложения, поэтому мы установим контекст в файле макета Администрирование (`src/layout/Admin.svelte`). Затем мы запустим его на первой странице, отрендеренной в этом разделе.

## Создание контекста трекера

Внутри файла `src/layout/Admin.svelte` в теге `script` добавьте следующий код:


```jsx
import {Tracker, key} from "../context/tracker"

let tracker = new Tracker({
    projectKey: __myapp.env.OPENREPLAY_PROJECT_KEY
});
  
setContext(key, {
 getTracker: () => tracker
} )
```

```jsx
import {key} from "../../context/tracker"
  
let {getTracker} = getContext(key)
onMount( () => {
	let tracker = getTracker()
	tracker.start()
})
```
