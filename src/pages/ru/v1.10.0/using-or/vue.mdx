Получение трекера для работы на приложении Vue довольно просто.

## Добавление кода отслеживания

Хотя вы можете [получить версию кода для копирования и вставки непосредственно из платформы при создании нового проекта], этот код слишком простой и затруднит настройку и добавление дополнительных плагинов.

Предполагая, что вы создаете SPA (приложение одностраничного дизайна), все, что вам нужно сделать, - это добавить следующий код в ваш скрипт настройки в основном файле.

<Aside type="tip">
**Примечание для пользователей самохостинга**: если вы используете самохостинговую версию OpenReplay, вам также необходимо указать свойство конфигурации `ingestPoint`.
Это свойство указывает на конечную точку ввода данных трекера. Пользователи облачного сервиса не нуждаются в этом,
поскольку по умолчанию трекер будет знать, где находится SaaS-версия этой конечной точки, но если вы используете самохостинг, вам нужно будет указать ее (это должно быть что-то вроде `https://openreplay.mydomain.com/ingest`).
</Aside>

### Обработка "projectKey" в вашем коде

Как примечание, «projectKey», который вы настраиваете, является тем, что предоставляется платформой OpenReplay.
Это значение не следует хранить в жестком коде в вашем коде по соображениям безопасности, а скорее на каком-либо файле/системе конфигурации.

Одним из вариантов будет добавление переменной окружения в файл `.env` в корне вашего проекта и префиксирование переменной с `VITE_`, что позволит вам использовать ее в вашем коде Vue.
Вы можете использовать ее позже, используя `import.meta.env.<your variable name>`.
Таким образом:
```html
<script setup>
//... your code here
const tracker = new Tracker({
  projectKey: "<your project key>"
});

tracker.start();
//... more code here
</script>
```
