Установка трекера OpenReplay в проекте на основе Gatsby довольно проста.

![Сессия воспроизведения сайта Gatsby](images/gatsby-ss.png)

Учитывая, что Gatsby использует React под капотом, все, что нам нужно сделать, это добавить код трекера в главную страницу и вызвать метод `start` с помощью хука `useEffect`.

Давайте посмотрим.

## Почему устанавливать трекер на статическом сайте?

Отсутствие динамического контента в вашем приложении не значит, что вы не хотите воспользоваться некоторыми другими функциями OpenReplay.

Хотя вы не будете использовать воспроизведение сеансов для поиска ошибок, вы сможете захватить интересные метрики и создать [собственные пользовательские панели инструментов](https://docs.openreplay.com/tutorials/custom-dashboard) вокруг этих метрик.

Вы можете использовать метрики производительности, веб-виталов и даже ресурсных метрик, чтобы получить общее представление о том, какой опыт пользователей с вашим сайтом.

## Настройка трекера внутри Gatsby

Учитывая статический сайт Gatsby и точку входа для него в файле `index.tsx` в папке `pages`, вы хотите добавить следующий код в него:


```jsx
import Tracker from '@openreplay/tracker';
const tracker = new Tracker({
  projectKey: process.env.GATSBY_OPENREPLAY_PROJECT_KEY
});

const Index = ({data, location }) => {

  React.useEffect(() => {
    tracker.start();
  }, [])

   //the rest of your code here...
}
```
