## Инициализация
При создании трекера OpenReplay вы можете предоставить несколько опций конфигурации, чтобы настроить многие аспекты записи и опыта записи.

Вы должны установить опцию `projectKey` в конструкторе. Это значение можно получить из панели управления OpenReplay в разделе «Настройки> Проекты».

## Опции инициализации

Есть набор опций, которые можно передать в конструктор. Только `projectKey` обязателен.

### Общее назначение

- `projectKey: string` ID проекта, который вы отслеживаете.
- `sessionHash?: string` Хэш начальной сессии. Это полезно, когда сессии пересекают разные поддомены вашего веб-приложения, но вы хотите сшить их в одну запись. В случае, если невозможно продолжить сессию (она не существует или завершена), трекер автоматически начнет новую. Он также возвращается на `stop()`. Более подробно об этом здесь.
- `ingestPoint?: string` Ваш домен OpenReplay (т. е. https://openreplay.mydomain.com/ingest), к которому трекер будет отправлять события. Для пользователей OpenReplay Cloud это необязательно. По умолчанию: `https://api.openreplay.com/ingest` (что указывает на OpenReplay Cloud).
- `revID?: string` Идентификатор ревизии вашего веб-приложения. Полезно при поиске проблем, возникающих на определенной версии релиза.
- `resourceBaseHref?: string` Относится к общедоступному домену, где могут быть извлечены активы (стили, шрифты и значки) OpenReplay. Поскольку они необходимы для правильной повторной передачи сеанса, эта опция полезна для обхода ограничен
```bash
npm i @openreplay/tracker
```
