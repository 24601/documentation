По умолчанию наш трекер отправит много полезной информации на нашу платформу, но все эти данные стандартны. Пока данные присутствуют в Web API, скорее всего, мы их отслеживаем.

Но что произойдет, если вы также хотите дополнить ваши записи сессий данными, специфичными для вашего приложения?

Например, понимание, под каким платным планом находится пользователь, когда вы просматриваете его повторное воспроизведение? Эта информация может дать дополнительный контекст вашим разработчикам, поэтому не стоит игнорировать ее просто потому, что она не стандартная.

Вот здесь и вступает в игру метаданные.

## Что такое метаданные?

В контексте записи сессии метаданные - это вся информация, которую пользователь не генерирует, но которая каким-то образом связана с этим пользователем.

Другими словами, если пользователь нажимает на определенную ссылку или выполняет определенное действие, это не метаданные, это пользовательское событие (проверьте [документацию о пользовательских событиях](/installation/custom-events), если вы хотите узнать больше о них).

Но если есть флаг, который говорит вам, что пользователь находится под бесплатным планом или платным. Или, возможно, код отслеживания, пок
```jsx
// inside the main component
const { startTracking, setMetadata } = useContext(TrackerContext)
//...
useEffect(() => {
    async function getProds() {
      await startTracking()
      setMetadata('plan', getPlan()) //addition 
      setMetadata('utm_source', getUTMSource()) //addition
      dispatch(getMakeUpProducts() as any)
    }

    getProds()
  }, [dispatch])
```
