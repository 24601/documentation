OpenReplay stack может быть установлен на одну машину, а OVHcloud Dedicated server, VPS, виртуальная машина Private Cloud или экземпляр Public Cloud - идеальный кандидат. Вот как это сделать.

## Закажите свой сервер или запустите свой экземпляр

Вы можете использовать любой из следующих продуктов OVHcloud для развертывания OpenReplay:

- Выделенный сервер
- VPS
- Private Cloud (внутри виртуальной машины)
- Экземпляр Public Cloud

Предварительные требования:

- Выберите *Ubuntu Server 20.04 Focal Fossa* в качестве операционной системы.
- Минимальные требования для машины, на которой запущен OpenReplay, составляют `2 vCPUs, 8 GB ОЗУ, 50 GB хранилища`, иначе службы OpenReplay просто не запустятся. Этого должно быть достаточно для низкого / умеренного объема. Если вы ожидаете высокой нагрузки, вы должны масштабировать отсюда.
- Общедоступный IP-адрес, указывающий на ваш сервер / экземпляр.

## Развертывание OpenReplay

1. Убедитесь, что ваш сервер / экземпляр запущен, а затем подключитесь к нему через SSH от имени root

2. Установите OpenReplay, указав домен, на котором он будет работать (например, DOMAIN_NAME = openreplay.mycompany.com):


```bash
sudo wget https://raw.githubusercontent.com/openreplay/openreplay/main/scripts/helmcharts/openreplay-cli -O /bin/openreplay 
sudo chmod +x /bin/openreplay
openreplay -i DOMAIN_NAME
```


## Настройка TLS / SSL

OpenReplay имеет дело с конфиденциальными данными пользователей и поэтому требуется HTTPS для работы. Это обязательно, иначе трекер просто не начнет запись. То же самое касается панели управления: без HTTPS вы не сможете повторить сеансы пользователей.

Вам потребуется предоставить (или сгенерировать) свой собственный SSL-сертификат.

1. Сначала перейдите в панель управления OVHcloud в «Web cloud»> «Доменные имена»> ваш домен (т. е. mycompany.com)> «DNS-зона» (или другой провайдер DNS-услуг) и создайте «A Record». Используйте домен, который вы ранее указали во время установки, и укажите на сервер / экземпляр с помощью его общедоступного IP-адреса.

2. Если вы приводите свой собственный сертификат, создайте секрет SSL, используя следующую команду: `kubectl create secret tls openreplay-ssl -n app --key="private_key
```yaml
ingress-nginx: &ingress-nginx

      ssl-redirect: true
```
