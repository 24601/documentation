منذ الإصدار 1.10.0، قمنا بإطلاق أداة CLI الخاصة بنا. باستخدامها، يمكن تنفيذ أهم المهام التي يجب تنفيذها لإدارة نسخة OpenReplay المضمنة بشكل محلي بسهولة باستخدام أمر واحد.

## تثبيت CLI

تأتي أداة CLI مع كل نسخة مضمنة محليا من OpenReplay.

## استخدام CLI

تتميز صيغة CLI بالأمر التالي:


```bash 
openreplay [command] <service>
```


كما يلي قائمة الأوامر المتاحة:

- `-h أو --help`: سيظهر هذا الأمر قائمة الأوامر والخدمات المتاحة.
- `-s أو --status`: إذا كنت ترغب في معرفة الحالة الحالية للخدمات المثبتة، فسيخبرك هذا الأمر كل ما تحتاج إليه.
- `-u أو --upgrade`: يستخدم لترقية التثبيت إلى أحدث إصدار متاح.
- `-U أو --deprecated-upgrade /path/to/old_vars.yaml`: ????
- `-r أو --restart`: يتيح لك إعادة تشغيل جميع الخدمات. بالفعل، يمكنك إصلاح الخدمات التي لا تستجيب.
- `-R أو --Reload`: يقوم بإعادة تحميل تكوين خدمة، لذلك استخدمه فقط عند تغيير شيء.
- `-p أو --install-packages`: يقوم بتحديث جميع التبعيات الخارجية (مثل `kubectl`، `helm`، إلخ).
- `-i أو --install domain.name.com`: يقوم بتثبيت OpenReplay على الجهاز.
- `-e أو --edit`: يقوم بتحرير التكوين المثبت لـ OpenReplay وإعادة التحميل.
- `-l أو --logs SERVICE`: يعرض السجلات من خدمة معينة.

كما يلي قائمة جميع الخدمات المتاحة لـ CLI:

- التنبيهات
- الأصول
- المساعدة
- Chalice
- قاعدة البيانات
- Ender
- الواجهة الأمامية
- التحليلات الحساسة
- HTTP
- التكاملات
- متحكم NGIX
- الأقران
- الحفر
- قارئ خرائط المصدر
- التخزين

## مثال

للحصول على حالة النشر، قم بتنفيذ الأمر CLI التالي:


```bash 
openreplay -s 
```
