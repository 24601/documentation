ربط معرف المستخدم الداخلي بتسجيل الجلسة التي يتم تسجيلها بمتابعة الخطوات التالية.

### مع NPM

إدخال معرف المستخدم عند بدء المتابع:


```js
const tracker = new OpenReplay({
  projectKey: PROJECT_KEY
});

tracker.start({
  userID: "john@doe.com",
  metadata: {
    balance: "10M",
    plan: "free"
  }
});
```


إذا لم يكن ذلك ممكنًا (يتعرف على معرف المستخدم لاحقًا في تدفق التنقل، لذلك بعد بدء المتابع)، فاستدعاء طريقة setUserID لتحديد مستخدميك. يمكن تغيير هوية المستخدم في أي وقت خلال الجلسة باستدعاء setUserID. ومع ذلك، سيحافظ OpenReplay على آخر معرف المستخدم المضاف.


```js
tracker.setUserID('john@doe.com'); // after tracker.start()
```


### مع كود JavaScript

بالأسفل هو عينة على كيفية إدخال معرف المستخدم باستخدام الكود. لاحظ المتغير / السطر startOpts لإدخال معرف المستخدم.


```js
<!-- OpenReplay Tracking Code -->
<script>
var initOpts = { projectKey: "GxPpaDARdn2345fgt321" };
var startOpts = { userID: getUserID() }; // use startOpts variable to inject userID
(function(A,s,a,y,e,r){
  r=window.OpenReplay=[e,r,y,[s-1, e]];
  s=document.createElement('script');s.src=A;s.async=!a;
  ...
})("//static.openreplay.com/latest/openreplay.js", 1, 0, initOpts, startOpts);
</script>
```


إذا لم يكن ذلك ممكنًا (يتعرف على معرف المستخدم لاحقًا في تدفق التنقل، لذلك بعد بدء المتابع)، فبدلاً من ذلك استدعاء طريقة setUserID لتحديد مستخدميك. يمكن تغيير هوية المستخدم في أي وقت خلال الجلسة باستدعاء setUserID. ومع ذلك، سيحافظ OpenReplay على آخر معرف المستخدم المضاف.

var startOpts = { userID: "" };
(function(A,s,a,y,e,r){
  r=window.OpenReplay=[e,r,y,[s-1, e]];
  s=document.createElement('script');s.src=A;s.async=!a;

})("//static.openreplay.com/latest/openreplay.js", 1, 0, initOpts, startOpts);
</script>
  ...
  OpenReplay.setUserID("john@doe.com"); // set the userID later in your code
