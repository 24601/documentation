منذ الإصدار 1.10.0، قمنا بإطلاق أداة CLI الخاصة بنا. باستخدامها، يمكن تنفيذ المهام الأكثر شيوعا التي يجب تنفيذها لإدارة نسخة OpenReplay المضمنة بشكل محلي باستخدام أمر بسيط.

## تثبيت CLI

تأتي أداة CLI مع كل نسخة مضمنة محليا من OpenReplay.

## استخدام CLI

تتميز أداة CLI بالصيغة التالية:


```bash 
openreplay [command] <service>
```


فهيئة الأوامر المتاحة هي:

- `-h أو --help`: سيظهر هذا الأمر قائمة الأوامر والخدمات المتاحة.
- `-s أو --status`: إذا كنت ترغب في معرفة الحالة الحالية للخدمات المثبتة، فسيعطيك هذا الأمر كل ما تحتاج إليه للمعرفة.
- `-u أو --upgrade`: يستخدم لترقية التثبيت إلى أحدث إصدار متاح.
- `-U أو --deprecated-upgrade /path/to/old_vars.yaml`: ????
- `-r أو --restart`: يتيح لك إعادة تشغيل جميع الخدمات. بالفعل، يمكنك إصلاح الخدمات التي لا تستجيب.
- `-R أو --Reload`: يعيد تحميل تكوين خدمة، لذلك استخدمه فقط عند تغيير شيء.
- `-p أو --install-packages`: يقوم بتحديث جميع التبعيات الثالثة (مثل `kubectl`، `helm`، إلخ).
- `-l أو --logs SERVICE`: يعرض السجلات من خدمة معينة.

فهيئة جميع الخدمات المتاحة لأداة CLI هي:

- التنبيهات
- الأصول
- المساعدة
- Chalice
- قاعدة البيانات
- Ender
- الواجهة الأمامية
- التحليلات الشاملة
- HTTP
- التكاملات
- Ngix-Controller
- الأقران
- الحوض
- Sourcemapreader
- التخزين

## مثال

للحصول على حالة التثبيت، قم بتنفيذ الأمر CLI التالي:


```bash 
openreplay -s 
```
